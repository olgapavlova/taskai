# `07` • Придумать сырьё

Первый шаг к придумыванию задачи — контринтуитивен. **Так редко делают.**

Нужно придумать, какое в задаче понадобится **сырьё**. Ощутимый кусок реального мира, без которого задача будет выхолощеной и вызывающе учебной. А с ним — интересной.

### Примеры сырья
1. Скриншот реального интерфейса — для задач на интерфейсный дизайн.
2. Техническое задание реального проекта — для задач на бизнес-аналитику.
3. Реальный договор реального проекта — для задач по проектному управлению.
4. Настоящие таблицы с данными какого-нибудь ведомства — для задач на визуализацию данных.
5. Письма саппорта или рекламный спам — для задач на digital-копирайтинг.

> [!CAUTION]
> **Сырьё дико сложно добывать.**\
> Захочется полениться и обойтись как-нибудь так.\
> Не надо.

**Мы ещё не придумываем задачу.** Только сырьё. Для каждой задачи, да.

**Мы ещё не добываем сырьё.** Только придумываем.

**Достаточно самого общего вида.** Скриншот — скриншот — скриншот — скриншот — …

**И снова тут появляется шанс на делегирование.** Вы сможете объяснить коллегам, какое сырьё вам нужно. И они принесут. Но это чуть позже.

## Короче, нужно:
- [ ] для каждой задачи придумать вид сырья, из которого вы будете её собирать
- [ ] записать эту свою потребность в сырье. Позадачно.

## Итог
Мы знаем, из каких кусков реального мира будем конструировать задачи. И сколько их (примерно).
